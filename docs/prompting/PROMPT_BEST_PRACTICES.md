# Prompt Best Practices

Use this checklist to get consistent, premium output from this repository.

## 1. Start With Outcome, Not Topic

Bad:

- "Make slides about AI agents."

Better:

- "Create an 8-slide LinkedIn carousel for engineering managers to adopt agent workflows with low risk."

## 2. Provide Intent + Audience + CTA

Always include:

- topic
- audience
- objective (`authority`, `education`, `lead_generation`, etc.)
- desired CTA

## 3. Ask for Visual Modes Explicitly

If you care about visual quality, state expectations:

- where KPI cards should appear
- where chart mode should appear
- where image mode should appear
- where panel should stay hidden

## 4. Keep Copy Constraints Explicit

Include constraints:

- 3 bullets per body slide preferred
- minimal text
- no duplicated wording across headline/intro/bullets/panel
- concise headline length

## 5. Require Research Boundaries

For changing topics:

- ask for current sources
- ask for explicit date context
- ask to separate verified facts vs inferred statements

## 6. Use Structured Placeholders

Use placeholders so AI can fill cleanly:

- `<TOPIC>`
- `<AUDIENCE>`
- `<OBJECTIVE>`
- `<SLIDE_COUNT>`
- `<KEY_POINTS>`
- `<CTA>`
- `<COLOR_THEME>`

## 7. Require Build Artifacts

Ask for exact deliverables:

- final content spec path
- PDF path
- validation report
- render intelligence log
- overseer reports

## 8. Prompt Anti-Patterns to Avoid

- vague "make it better" without audience/objective
- forcing every slide to have same structure
- requesting too many ideas per slide
- asking for visuals without specifying relevance

## 9. Golden Prompt Pattern

Use this scaffold:

1. Objective and audience
2. Topic and non-goals
3. Slide count and structure
4. Visual expectations
5. Copy constraints
6. Research constraints
7. Output artifact requirements
